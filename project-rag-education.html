Overview

One Problem, Two Entry Points

Clinical patient education fails in two distinct places: at the bedside, where generic leaflets don't fit the individual patient's diagnoses and medications; and after discharge, where patients forget what nurses explained and have no easy way to look it up. This project builds one shared knowledge base and two systems that serve each gap.

System A · Public Portal

北醫衛教小幫手 — Patient-Facing Query Portal

A publicly deployed web application that lets patients and families query hospital-approved education content after discharge — no login, no personal data needed. Designed for the moment a patient gets home and can't remember what the nurse said.

View Live Site

Knowledge Base

210+ hospital-approved leaflets — chronic disease, wound care, maternal health, paediatrics, oncology, post-discharge instructions.

Query Interface

Natural language input in Mandarin. Supports voice input (國語/台語) for accessibility. No personal data required at any step.

Grounded Answers

Every response cites the source leaflet. Powered by Vertex AI RAG — no free-form generation without document grounding.

Knowledge Gap Loop

When the system can't answer, users submit the question topic — logged as a knowledge base expansion queue for clinical team review.

Video Integration

Links to TMUH YouTube health education channels for topics where video is more effective (injection technique, physical therapy).

Privacy by Design

No login, no name, no patient ID required. Anonymous query throughout — safe for post-discharge public access.

System B · Personalised Generation

Agentic RAG Pipeline — EHR-Driven Education Reports

A clinical-facing multi-agent pipeline that reads a patient's EHR, identifies relevant education needs, and generates a personalised citation-grounded report — using only hospital-approved documents, never free-form LLM knowledge.

7-Stage Pipeline

Stage 0

Knowledge Base Preparation

Chunk and standardise 225 leaflets into a structured, searchable clinical knowledge base.

Stage 1

Clinical Feature Extraction

Extract ICD-10 diagnoses and prescriptions from MIMIC-IV EHR per patient.

Stage 2

Candidate Retrieval Agent

LLM agent selects candidate leaflets using diagnoses, prescriptions, and semantic clinical reasoning — not fixed top-K similarity.

Stage 3

Relevance Verification Agent

Second agent evaluates clinical relevance and coverage completeness — checks whether comorbidity-specific content is actually present, not just keyword-matched.

Stage 4

Coverage Assessment & Extended Search

Detects unmet educational needs and performs targeted additional retrieval for uncovered comorbidities or medications.

Stage 5

Grounded Multi-LLM Generation

Master generation agent produces citation-grounded personalised summaries from verified leaflets only. RAG-only constraint active throughout.

Stage 6

Report Output

Structured PDF / Word report with per-claim traceable citations and explicit tracking of unresolved questions for clinician review.

Key design choice: multi-agent verification replaces fixed top-K retrieval. Each agent has a distinct role — retrieval, relevance check, coverage assessment, generation — which catches comorbidity gaps that single-pass retrieval misses.

Future Work

Next Directions

  • Hallucination mitigation (System B): Per-claim source verification and section-level grounding constraints to reach the <5% clinical safety threshold.
  • NIS integration (System B): Connect to nursing information system to incorporate fall-risk, pressure ulcer, and discharge readiness assessments into the patient profile.
  • Voice interface (System A): Expand STT to full conversational interaction for elderly or low-literacy patients, beyond current single-query voice input.
  • Knowledge gap feedback loop (both): Unresolved questions from both systems are collected as structured signals for knowledge base expansion — reviewed by clinical team before integration.
  • Unified architecture: Merge the two systems so that a patient's discharge triggers a pre-generated personalised education set in System B, which becomes their default view in System A post-discharge.
